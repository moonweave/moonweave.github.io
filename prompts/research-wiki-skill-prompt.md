# 메인 프롬프트

아래 프롬프트를 Codex 또는 Claude Code에 붙여넣고, 내 Obsidian 볼트 경로를 함께 알려주세요.

```text
내 Obsidian Research Wiki를 Codex 또는 Claude Code에서 불러 쓸 수 있는 AI 에이전트용 스킬로 바꿔줘.

목표는 단순합니다.

내 볼트를 AI가 직접 탐색하고,
논문 노트 / 아이디어 노트 / 질문 노트 / 정리 노트를 구분해서,
필요할 때 근거 파일을 기반으로 답하게 만드는 것입니다.

전편의 Obsidian Research Wiki 설계 프롬프트로 만든 볼트라면, 그 구조를 우선 존중해줘.
기존 볼트를 새 구조로 갈아엎는 것이 아니라, AI가 이미 있는 구조를 잘 찾아 쓰게 만드는 것이 목표입니다.

볼트 경로:
[여기에 내 Obsidian 볼트 경로를 입력]

원하는 스킬 이름:
[예: research-wiki]

스킬 이름 규칙:
- 영어 소문자, 숫자, 하이픈만 사용해줘.
- 공백, 한글, 특수문자는 쓰지 마.
- "claude", "anthropic" 같은 특정 회사명은 이름에 넣지 마.

먼저 해야 할 일:

1. 실제 볼트 구조 확인
- 경로가 열리는지 확인해줘.
- 파일과 폴더를 실제로 보고 판단해줘.
- 폴더 이름만 보고 과하게 단정하지 말고, 대표 파일 몇 개를 열어 구조를 확인해줘.
- 파일 접근 권한이 없으면, 내가 붙여넣을 수 있는 폴더 트리와 예시 노트 양식을 먼저 요청해줘.

2. 볼트 역할 추정
- 어떤 폴더가 논문 노트인지 확인해줘.
- 어떤 폴더가 아이디어 노트인지 확인해줘.
- 어떤 폴더가 질문, 리서치 로그, 작업 기록인지 확인해줘.
- 태그, 파일명 규칙, 링크 규칙이 있다면 반영해줘.
- 확실하지 않은 부분은 확정하지 말고 "추정" 또는 "확인 필요"로 표시해줘.

3. 스킬 설계
이 볼트를 AI가 사용할 때 필요한 규칙을 설계해줘.

반드시 포함할 규칙:
- 저장된 노트 내용과 AI의 추론을 구분한다.
- 논문 원문 내용, 내 메모, 아이디어를 섞어서 확정 사실처럼 말하지 않는다.
- 답변할 때 가능한 한 근거 파일 경로를 남긴다.
- 근거가 부족하면 부족하다고 말한다.
- 처음부터 볼트 전체를 훑지 말고, 작은 검색에서 시작해 필요할 때만 범위를 넓힌다.
- 사용자의 질문이 모호하면 먼저 검색 기준을 좁힌다.
- 검색 결과가 없으면 비슷한 키워드로 한 번 더 시도하되, 없는 내용을 만들어내지 않는다.

4. SKILL.md 작성
아래 형식으로 바로 저장 가능한 SKILL.md를 작성해줘.

필수 포함:
- YAML frontmatter
  - name
  - description
- 언제 이 스킬을 사용할지
- 어떤 환경에서만 쓸 수 있는지
- 볼트 경로
- 먼저 확인할 폴더
- 검색 순서
- 답변 규칙
- 근거 표시 규칙
- 근거 부족 시 행동
- 사용 예시 질문

주의:
- 과하게 복잡한 스크립트나 자동화를 만들지 마.
- 현재 볼트 구조만으로 바로 쓸 수 있는 최소 스킬을 먼저 만들어줘.
- 실제 파일 산출물은 우선 SKILL.md 하나로 제한해줘. README, 설치 가이드, 보조 문서는 만들지 마.
- 불확실한 부분은 TODO로 남겨줘.
- 로컬 파일 접근이 가능한 환경에서만 제대로 동작한다고 명시해줘.
- 비공개 노트, 미공개 아이디어, 개인정보가 외부로 노출되지 않도록 주의 문구를 넣어줘.
- 특정 에이전트 전용 기능에 과하게 의존하지 말고, Codex와 Claude Code 둘 다 이해할 수 있는 형태로 작성해줘.
- 설치 위치는 사용하는 도구에 따라 달라질 수 있으니, Codex와 Claude Code 각각에 대해 "예시 경로"와 "확인 필요"를 분리해서 안내해줘.

출력 형식:

1. 볼트 구조 요약
2. 스킬 설계 방향
3. 최종 SKILL.md
4. 설치 또는 적용 방법
5. 테스트 질문 5개
6. TODO / 확인이 필요한 부분
```
